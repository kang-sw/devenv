# claude-dash — Interactive worktree-scoped Claude TUI multiplexer

## Context

**Ticket:** `ai-docs/tickets/todo/260426-feat-claude-dash.md` (4 phases).
**Spec stems:** `260426-claude-dash`, `260426-claude-dash-layout`,
`260426-claude-dash-worktree-tabs`, `260426-claude-dash-interactive-terminal`,
`260426-claude-dash-named-agents`, `260426-claude-dash-process-lifecycle`,
`260426-claude-dash-keybindings` (all in `ai-docs/spec/tools.md`).

**Sibling project:** `tools/claude-watch/` is the existing read-only viewer.
Phase 3 ports `parser.rs` + `renderer.rs` + `session.rs` (`SessionEntry`,
`parse_session_metadata`, `find_agent_name`) from claude-watch. The agreed
convention is **copy-then-share-later**: phase 3 copies the three modules
verbatim into `tools/claude-dash/src/`. A future refactor may extract a
shared `claude-jsonl` library; do not pre-empt that here.

**Termwiz API discovery (do not re-research):**

- `termwiz::terminal::buffered::BufferedTerminal` requires a live `Terminal`
  target — unusable for off-screen modeling. Use `termwiz::surface::Surface`
  directly. The ticket's "or equivalent screen model" wording authorizes this.
- `Surface` accepts `Change` enum variants via `add_change()` /
  `add_changes()`. It does **not** consume raw bytes.
- `termwiz::escape::parser::Parser::parse(bytes, |action| ...)` decodes a byte
  slice into `termwiz::escape::Action` events. The bridge from `Action` to
  `Change` (and direct cell mutations for control sequences with no `Change`
  equivalent) must be implemented by us in `vt.rs` — termwiz does not ship
  this bridge.
- `Surface::screen_lines() -> Vec<Cow<Line>>` returns the current cell grid;
  each `Line` exposes per-cell `CellAttributes` (fg, bg, bold, italic,
  reverse, underline). This is the read API for ratatui rendering.

**portable-pty 0.9 API (do not re-research):**

- `portable_pty::native_pty_system() -> Box<dyn PtySystem>`.
- `pty_system.openpty(PtySize { rows, cols, pixel_width: 0, pixel_height: 0 })
  -> Result<PtyPair>` returns `{ master, slave }`.
- `pair.slave.spawn_command(CommandBuilder) -> Result<Box<dyn Child + Send + Sync>>`.
- `pair.master.try_clone_reader() -> Result<Box<dyn Read + Send>>` — the
  reader is **blocking**. Move it to a dedicated thread.
- `pair.master.take_writer() -> Result<Box<dyn Write + Send>>` — single-shot;
  call once per PTY and store the writer.
- `pair.master.resize(PtySize)` issues the kernel `TIOCSWINSZ` ioctl.
  Signature is `resize(&self, PtySize) -> anyhow::Result<()>` — takes
  `&self`, not `&mut self`.
- `child.try_wait() -> IoResult<Option<ExitStatus>>` is non-blocking;
  poll each frame. `PtySession::try_wait` wraps this as
  `child.try_wait().ok().flatten()` to expose `Option<ExitStatus>`.
- `child.kill() -> IoResult<()>` — the `Drop` impl calls
  `let _ = self.child.kill();` to suppress the result.

**ratatui rendering decision:** use direct `Buffer` cell writes via
`Widget::render` (or a closure in `Frame::render_widget`). The `Canvas`
widget targets braille/dot drawing and is wrong here. The VT widget walks
`Surface::screen_lines()` and writes each cell to the inner `Rect` via
`buf[(x, y)].set_symbol(&str).set_style(Style)`.

**Hotkey priority (locked by ticket constraint):**
`Ctrl+[`, `Ctrl+]`, `Ctrl+1`–`Ctrl+9`, `Ctrl+Q` are app-only and never
forwarded to the PTY. All other keys, when the main slot is active, route
to the PTY writer encoded as VT byte sequences (see step 4.4 below).

**Worktree directory naming (claude-watch precedent):** the
`~/.claude/projects/<escaped>` formula replaces `/` with `-` in the
worktree path. Reference `tools/claude-watch/src/session.rs::discover_project_dirs`.

**Active ws-worktree marker:** the ticket directs auto-spawn for any
worktree directory present under `.claude/worktrees/<name>`. This
sub-directory contains active ws-framework worktrees; its presence is the
auto-spawn signal. Worktrees not under this path spawn on first selection.

#### 1.3 `pty.rs` — PTY lifecycle

Public contract:

```rust
pub struct PtySession {
    master: Box<dyn portable_pty::MasterPty + Send>,
    child: Box<dyn portable_pty::Child + Send + Sync>,
    writer: Box<dyn std::io::Write + Send>,
    rx: std::sync::mpsc::Receiver<Vec<u8>>,
    size: portable_pty::PtySize,
    reader_thread: Option<std::thread::JoinHandle<()>>,
}

impl PtySession {
    pub fn spawn(cwd: &std::path::Path, size: portable_pty::PtySize)
        -> anyhow::Result<Self>;
    pub fn write(&mut self, bytes: &[u8]) -> std::io::Result<()>;
    pub fn try_recv_chunk(&self) -> Option<Vec<u8>>; // drain helper iterates
    pub fn resize(&mut self, size: portable_pty::PtySize) -> anyhow::Result<()>;
    pub fn try_wait(&mut self) -> Option<portable_pty::ExitStatus>;
    pub fn kill(&mut self);
    pub fn size(&self) -> portable_pty::PtySize;
}
```

**Spawn sequence:**

1. `let pty_system = portable_pty::native_pty_system();`
2. `let pair = pty_system.openpty(size)?;`
3. Build `CommandBuilder::new("claude")`. Set `cwd` via `cmd.cwd(cwd)`.
   Inherit `PATH` and other env vars; do not strip them.
4. `let child = pair.slave.spawn_command(cmd)?;`
5. `let writer = pair.master.take_writer()?;`
6. `let reader = pair.master.try_clone_reader()?;`
7. Spawn a thread that loops `reader.read(&mut buf)`. On every read, send
   `Vec<u8>` of the read slice to a `std::sync::mpsc::channel`. Exit the
   thread on `Ok(0)` (EOF) or any read error.
8. **Drop the slave** (`drop(pair.slave)`) immediately after spawning the
   child. WezTerm's docs require this so the child is the only slave-side
   holder; otherwise `read` on master will not return EOF when the child
   exits.
9. Store `master`, `child`, `writer`, `rx`, `size`, and the reader-thread
   `JoinHandle` on `PtySession`.

**Channel sizing:** unbounded `std::sync::mpsc::channel`. Bounded would
deadlock the reader thread on bursts; the main loop drains every frame.

**Drain pattern:** main loop calls `pty.rx.try_recv()` in a `while let Ok`
loop until empty, concatenating chunks into one `Vec<u8>` per frame, then
feeding that to `VtScreen::feed`. Drain after `crossterm::event::poll` and
before draw.

**Kill on drop:** `Drop for PtySession` calls `child.kill()` to enforce
"subprocesses terminate when claude-dash exits" (ticket Decision).

#### 1.4 `vt.rs` — VT screen state

Public contract:

```rust
pub struct VtScreen {
    surface: termwiz::surface::Surface,
    parser: termwiz::escape::parser::Parser,
    cols: u16,
    rows: u16,
}

impl VtScreen {
    pub fn new(cols: u16, rows: u16) -> Self;
    pub fn feed(&mut self, bytes: &[u8]);
    pub fn resize(&mut self, cols: u16, rows: u16);
    pub fn render_into(&self, buf: &mut ratatui::buffer::Buffer,
                       area: ratatui::layout::Rect);
    pub fn dimensions(&self) -> (u16, u16);
}
```

**`feed` implementation:** use `self.parser.parse_as_vec(bytes)` which
returns `Vec<Action>` directly (available in termwiz 0.23), avoiding the
borrow-checker issue with `parse`'s closure needing `&mut self`:

```rust
let actions = self.parser.parse_as_vec(bytes);
for action in actions { self.handle_action(action); }
```

The manual collect pattern also works (`parse` + closure pushing into a
pre-allocated Vec) but `parse_as_vec` is cleaner.

**Action → Surface translation (`handle_action`):** the minimum set
needed for `claude` interactive output (which uses ratatui itself, so
mostly CSI cursor moves + SGR + erase + plain print):

| `termwiz::escape::Action` variant | Translation |
|---|---|
| `Print(c)` | `surface.add_change(Change::Text(c.to_string()))` |
| `PrintString(s)` | `surface.add_change(Change::Text(s))` |
| `Control(ControlCode::LineFeed)` | `Change::Text("\n".into())` |
| `Control(ControlCode::CarriageReturn)` | `Change::Text("\r".into())` |
| `Control(ControlCode::Backspace)` | move cursor left by 1 via `Change::CursorPosition` |
| `Control(ControlCode::HorizontalTab)` | `Change::Text("\t".into())` |
| `CSI(Csi::Cursor(...))` | translate to `Change::CursorPosition { x: Position::Absolute, y: Position::Absolute }` based on the cursor variant |
| `CSI(Csi::Sgr(sgr))` | apply `Change::Attribute(AttributeChange::...)` per SGR field (fg, bg, intensity, italic, underline, reverse) |
| `CSI(Csi::Edit(EraseInLine(_)))` | `Change::ClearToEndOfLine(...)` (or full-line clear via cursor positioning + clear) |
| `CSI(Csi::Edit(EraseInDisplay(_)))` | `Change::ClearToEndOfScreen(...)` or `Change::ClearScreen` |
| `CSI(Csi::Mode(SetMode(...)))` | track `?1049` (alt-screen) and `?25` (cursor visibility) — ignore others; phase 1 acceptable behaviour |
| `OperatingSystemCommand(SetWindowTitle(_))` | ignore (no use case) |
| All other `Action` variants | log + ignore |

Use `Surface`'s public `add_change` and `Position::Absolute` /
`AttributeChange` types from `termwiz::surface`. The translator's job is
exhaustive coverage of `claude`'s output; missed variants degrade to
no-op (no panic). Mark unsupported variants as `#[allow(unreachable_patterns)]`
and document in the module header that termwiz updates may add new
variants.

**`render_into` implementation:** iterate `surface.screen_lines()`.
`screen_lines()` returns `Vec<Cow<Line>>`; use `.iter().enumerate()` for
the row index. `line.visible_cells()` returns `Iterator<Item = CellRef>`;
use `cell.cell_index() as u16` for the column index (not `.enumerate()`,
which gives sequential index and mishandles double-width cells). Pattern:

```rust
for (y, line) in surface.screen_lines().iter().enumerate() {
    let y = y as u16;
    for cell in line.visible_cells() {
        let x = cell.cell_index() as u16;
        // bounds check, style translate, set cell
    }
}
```

For each `(y, line)` with `y as u16` and each `cell` with `x = cell.cell_index() as u16`:

1. If `area.x + x >= area.right()` or `area.y + y >= area.bottom()`, skip.
2. Translate `cell.attrs()` (a `&CellAttributes`) into `ratatui::style::Style`:
   - `attrs.foreground()` → `Style::default().fg(...)` (map
     `ColorAttribute::TrueColorWithDefaultFallback(srgb)` → extract bytes
     with `let (r,g,b,_) = srgb.to_srgb_u8();` → `Color::Rgb(r,g,b)`,
     `ColorAttribute::TrueColorWithPaletteFallback(srgb, _)` → same
     conversion via `to_srgb_u8()` — discard the palette fallback,
     `ColorAttribute::PaletteIndex(n)` → `Color::Indexed(n)`,
     `ColorAttribute::Default` → no-op).
     (`SrgbaTuple` stores f32 components 0.0–1.0; `to_srgb_u8()` returns
     `(u8, u8, u8, u8)` with the 4th being alpha — discard it.)
   - Same for `attrs.background()`.
   - `attrs.intensity() == Intensity::Bold` → `Modifier::BOLD`.
   - `attrs.italic()` → `Modifier::ITALIC`.
   - `attrs.underline() != Underline::None` → `Modifier::UNDERLINED`.
   - `attrs.reverse()` → `Modifier::REVERSED`.
3. Set the cell: `buf[(area.x + x, area.y + y)].set_symbol(cell.str()).set_style(style)`.

**Cursor rendering:** read `surface.cursor_position()` and
`surface.cursor_visibility()`. When visible, call
`buf[(area.x + cx, area.y + cy)].set_style(Style::default().add_modifier(Modifier::REVERSED))`
on top of the cell. Do not move the user's terminal cursor — it stays at
the bottom-right (managed by ratatui).

**`resize` implementation:** `self.surface.resize(cols as usize, rows as usize)`.
Surface preserves cell content within the smaller bounding box and pads
the larger box with default cells. Update `self.cols`, `self.rows`.

#### 1.6 `ui.rs` — phase-1 layout

Single full-screen widget. The main panel is the entire frame area minus
1 row top reserved for a placeholder bar (will become the tab bar in
phase 2). For phase 1, render an empty top bar (just a horizontal line)
and dedicate the rest to the VT panel.

Phase-1 layout:

```
Layout::vertical([Constraint::Length(1), Constraint::Min(0)])
```

#### 1.7 `main.rs` — phase-1 event loop

Reuse `claude-watch`'s panic-hook + `EnableMouseCapture` pattern. Mouse
capture is **off** in phase 1 — `claude` needs raw mouse passthrough only
in later phases if at all; do not enable it yet. Constants:

```rust
const FRAME_MS: u64 = 10;       // 10ms tick — keep PTY responsive.
const POLL_MS: u64 = 5;         // crossterm::event::poll timeout.
```

Event loop:

```
loop:
  1. Drain crossterm events via `while event::poll(Duration::from_millis(POLL_MS))?`
     - dispatch each event to a `handle_event(&mut app, event)` function.
  2. Drain PTY output: `if let Some(ref s) = app.session {
       while let Some(chunk) = s.try_recv_chunk() { app.vt.feed(&chunk); }
     }`. (`app.session` is `Option<PtySession>`; guard before calling methods.)
  3. Poll process exit: `if app.exited_modal.is_none() {
       if let Some(ref mut s) = app.session {
         if let Some(status) = s.try_wait() {
           app.exited_modal = Some(ExitedModal { status });
         }
       }
     }`.
  4. Compute panel size from `terminal::size()` minus top-bar/border rows.
     If different from `app.last_panel_size`, call
     `app.vt.resize(cols, rows)` and
     `if let Some(ref mut s) = app.session { s.resize(PtySize { ... })?; }`.
  5. Draw via `terminal.draw(|f| ui::draw(f, &mut app))`.
  6. If app.should_quit, break.
  7. `std::thread::sleep(Duration::from_millis(FRAME_MS))`.
```

**Resize check location:** in step 4, before draw. The check uses
`terminal::size()` for an approximation; the authoritative size is the
inner rect of the VT block, which is only known inside `ui::draw`. Use
the same pattern as claude-watch: cache last panel inner dimensions in
`App` from inside `ui::draw`, then perform the `pty.resize()` call at
step 4 of the next iteration. One frame of size lag is acceptable.

**Phase-1 hotkey table (handled in `handle_event`):**

| Key | Action |
|---|---|
| `Ctrl+Q` | `app.should_quit = true`. |
| Modal active + `R` | drop session, spawn new in same cwd, clear `exited_modal`. |
| Modal active + `Q` | `app.should_quit = true`. |
| Any other key when modal inactive | encode and forward to PTY (see 1.8). |

#### 1.8 Key encoding for PTY forwarding

Translate `crossterm::event::KeyEvent` to VT byte sequences. Phase-1
minimum coverage:

| KeyCode | Bytes |
|---|---|
| `Char(c)` no modifiers | UTF-8 bytes of `c` |
| `Char(c)` with `Ctrl` (a–z) | single byte `c & 0x1f` |
| `Enter` | `\r` (0x0D) |
| `Backspace` | `0x7f` |
| `Tab` | `\t` |
| `BackTab` | `\x1b[Z` |
| `Esc` | `\x1b` |
| `Up` / `Down` / `Right` / `Left` | `\x1b[A` / `B` / `C` / `D` |
| `Home` / `End` | `\x1b[H` / `\x1b[F` |
| `PageUp` / `PageDown` | `\x1b[5~` / `\x1b[6~` |
| `Delete` | `\x1b[3~` |
| `F(n)` for n=1..=12 | xterm sequences (`\x1bOP` for F1, etc.) |

Forward via `app.session.write(&bytes)?`. Encoding lives in a free
function `pty::encode_key(KeyEvent) -> Option<Vec<u8>>`.

### Phase 2 — Worktree tab bar

#### 2.1 `worktree.rs` — discovery

```rust
pub struct Worktree {
    pub path: std::path::PathBuf,
    pub name: String,           // last path component
    pub is_active_ws: bool,     // dir exists under .claude/worktrees/<name>
    pub is_removed: bool,       // not in latest `git worktree list` but tab kept open
}

pub fn discover_worktrees() -> Vec<Worktree>;
```

**`discover_worktrees` logic:**

1. Run `git worktree list --porcelain`. Parse lines starting with
   `worktree ` to collect absolute paths.
2. For each path, derive `name = path.file_name().to_string_lossy()`.
3. `is_active_ws` test: read repo root from any worktree (the first
   `worktree` line is the main checkout). If
   `<repo_root>/.claude/worktrees/<name>` is a directory, set true.
4. `is_removed` is always false here; the App tracks removal state from
   poll-cycle deltas (see 2.4).
5. Order: main checkout first, then alphabetical by `name`.

#### 2.2 `app.rs` — multi-tab state

```rust
pub struct WorktreeTab {
    pub worktree: Worktree,
    pub session: Option<PtySession>,    // None until first activation, or after exit
    pub vt: VtScreen,                    // always exists; resized on activation
    pub last_inner_size: (u16, u16),
    pub exited_modal: Option<ExitedModal>,
}

pub struct App {
    pub tabs: Vec<WorktreeTab>,
    pub active_tab: usize,
    pub should_quit: bool,
    pub last_worktree_poll: std::time::Instant,
}
```

`App::new()` calls `discover_worktrees()`, builds tabs, then for every
tab where `is_active_ws` is true spawns a `PtySession` immediately.
Other tabs leave `session: None`; they spawn on first activation.

`App::activate_tab(idx)`: sets `active_tab = idx`. If
`tabs[idx].session.is_none() && !tabs[idx].worktree.is_removed`, spawn
the PTY now.

#### 2.3 `ui.rs` — tab bar

Top row layout becomes a tab strip rendered with `ratatui::widgets::Tabs`
(or manual `Line` rendering for finer control). Tab labels are
`tab.worktree.name`. Active tab uses
`Style::default().add_modifier(Modifier::REVERSED)`. Removed tabs append
` [removed]` (phase 4 detail; phase 2 may emit if state set).

Layout:

```
Layout::vertical([
    Constraint::Length(1),  // tab bar
    Constraint::Min(0),     // main panel
])
```

The active tab's `vt` and modal state drive the main panel. Render only
the active tab's VT — background tabs accumulate VT state but are not
drawn.

#### 2.4 Polling worktrees

Constant: `const WORKTREE_POLL_SECS: u64 = 5;` (matches ticket).

In the main loop after PTY-output drain:

```
if app.last_worktree_poll.elapsed() >= Duration::from_secs(WORKTREE_POLL_SECS):
    let new_list = discover_worktrees();
    app.reconcile_worktrees(new_list);
    app.last_worktree_poll = Instant::now();
```

`App::reconcile_worktrees(new: Vec<Worktree>)`:

1. For each new entry not present in `app.tabs` (matched by absolute
   path), append a new tab with `session: None`. Auto-spawn if
   `is_active_ws`.
2. For each existing tab whose path is missing from `new`:
   - If `session.is_some()`, mark `worktree.is_removed = true` (keep tab).
   - If `session.is_none()`, drop the tab entirely.
3. For tabs whose `is_removed` is already true and whose
   `session.try_wait().is_some()` (process dead), drop the tab.
4. Adjust `active_tab` if its index moved or its tab was dropped.

#### 2.5 Hotkey routing

Phase-2 hotkeys (handled before PTY forwarding):

| Key | Action |
|---|---|
| `Ctrl+[` | `app.active_tab = app.active_tab.saturating_sub(1)`; activate. |
| `Ctrl+]` | `app.active_tab = (app.active_tab + 1).min(tabs.len()-1)`; activate. |
| `Ctrl+Q` | quit. |

Note: crossterm reports `Ctrl+[` as `KeyCode::Char('[')` with
`KeyModifiers::CONTROL` on most terminals — but `Ctrl+[` is the same
byte as `Esc` (0x1b) and many terminals do not distinguish. Implementer
must verify on the target terminal (iTerm, Apple Terminal). **Fallback:**
if the target terminal collapses `Ctrl+[` to `Esc`, add an alternate
binding. Document the chosen binding in the README/help text.

Resolution rule: prefer the key event with `KeyModifiers::CONTROL` set if
crossterm distinguishes; otherwise reserve `Esc` for tab-prev (with a
note that this conflicts with PTY-forwarded Esc — known limitation, see
ticket Constraints).

The full hotkey set (`Ctrl+1`–`Ctrl+9`, `Ctrl+Q`, `Ctrl+[`, `Ctrl+]`)
short-circuits `pty::encode_key`. All other keys forward when the main
slot is active. `Ctrl+1` selects main slot; `Ctrl+2`–`9` are a no-op in
phase 2 (handled in phase 3).

#### 2.6 Per-tab VT sizing

Each `WorktreeTab` carries its own `VtScreen`. On tab activation, if
`vt.dimensions()` differs from the current main-panel inner size,
`vt.resize(...)` and `session.resize(...)`. Inactive tabs do not resize
until activated — they continue feeding bytes at their last size.

Background PTY drain: every frame, drain all tabs' channels (not just
the active one) and feed each tab's `VtScreen`. The session keeps
running in the background; only rendering is gated by `active_tab`.

#### 2.7 Phase-2 deliverable check

Manual: launch in a multi-worktree repo. Tab bar shows all worktrees.
`Ctrl+]` cycles forward, `Ctrl+[` backward. Each tab has independent
`claude` state. Creating a new worktree externally adds a tab within
~5s. Removing a worktree externally either removes the tab (if no
session) or marks it removed.

#### 3.1 Port `parser.rs`, `renderer.rs`, `session.rs` from claude-watch

- `parser.rs`: keep as-is (`Turn`, `ContentItem`, `parse_turns`).
- `renderer.rs`: keep as-is (`render_turns`, `RenderOptions`).

Add `pulldown-cmark = "0.13"` to `Cargo.toml` (renderer dependency).

#### 3.2 `agent.rs` — named-agent discovery

```rust
pub struct NamedAgent {
    pub name: String,                       // file stem of <name>.json
    pub uuid: String,                       // session UUID from JSON
    pub session_path: std::path::PathBuf,   // ~/.claude/projects/.../<uuid>.jsonl
    pub mtime: chrono::DateTime<chrono::Local>,
    pub token_total: Option<u64>,
}

pub fn discover_named_agents(git_root: &std::path::Path) -> Vec<NamedAgent>;
```

**Discovery logic:**

1. `agents_dir = git_root.join(".git").join(format!("ws@{}",
   git_root.file_name())).join("agents")`.
2. For each `*.json` in the directory: read, parse with `serde_json`.
   Required field: `"uuid": "<uuid>"`. Optional: ignore everything else.
3. Locate the session file: walk the **current worktree's**
   `~/.claude/projects/<escaped-cwd>/<uuid>.jsonl`. If absent, fall
   back to scanning all worktree project dirs (use the same
   `discover_project_dirs` helper from claude-watch's `session.rs`,
   ported as a shared function).
4. mtime from `fs::metadata`.
5. Sort by mtime descending.
6. Truncate to 8 entries (slot Ctrl+2..Ctrl+9).

**Token total:** parse each session via `parse_session_metadata` on a
background thread, with the same channel pattern claude-watch uses
(`work_tx` / `result_rx`, `parse_queued` flag in `NamedAgent`). Phase-4
detail polishes display; phase 3 may show `None` initially.

```
    Constraint::Length(1),  // agent slot row
    Constraint::Min(0),     // main panel
```

**Agent slot row format:**


**Main panel content:**

- `SlotKind::Main` → render the VT (same as phase 2).
- `SlotKind::Named(i)` → render the JSONL viewer using

#### 3.6 Hotkey routing — phase-3 additions

| Key | Action |
|---|---|
| `Ctrl+1` | `tab.active_slot = SlotKind::Main`; clear `agent_view`. |
| `Ctrl+2`..`Ctrl+9` | `tab.active_slot = SlotKind::Named(n-2)` if index exists; create `AgentView`. |
| `Up` / `Down` / `j` / `k` / `PageUp` / `PageDown` (in `Named` slot) | scroll the agent view (port from claude-watch). |
| All other keys (in `Main` slot) | forward to PTY (existing). |

Locked rule: when `active_slot == Main` and the key is not an app hotkey,
forward to PTY. When `active_slot == Named(_)`, scroll keys are consumed
by the viewer; other keys are ignored (do not forward to background PTY
— the user is reading, not driving).

displays the agent's JSONL turns. `Ctrl+1` returns to PTY. Switching
back to `Ctrl+2` after an agent runs more turns reflects the new content
within ~5s.

### Phase 4 — Process lifecycle polish

#### 4.1 Exit modal — keep tab open variant

Phase-1 exit modal already handles `[R] Restart` and `[Q] Quit-app`.
Phase 4 changes the second action to `[X] Close tab`:

| Modal key | Action |
|---|---|
| `R` | drop session, spawn fresh in same cwd, clear modal. |
| `X` | drop the entire tab (`app.tabs.remove(idx)`); re-anchor `active_tab`. |
| `Ctrl+Q` | quit app (still works globally). |

If the worktree is **also removed** (`is_removed: true`), the modal
shows only `[X] Close tab` — no restart, since the cwd no longer exists.

#### 4.2 Worktree-removal indicator

Tab label rendering: when `tab.worktree.is_removed`, append ` [removed]`
in `Color::Red`. The tab persists; reconciliation closes it only when
the process exits (handled in 2.4 step 3, which already covers this).

If the user activates a removed tab, no PTY restart is allowed — the
modal shown after process exit hides `[R]` for removed tabs (see 4.1).

#### 4.3 Token count display in agent slot row

Update the slot label format:

```
[1:main]  [2:<name1> 12.3k]  [3:<name2> 4.5k]  ...
```

Copy `format_tokens` from `claude-watch/src/ui.rs` into `claude-dash/src/ui.rs`
(or a shared helper location such as `agent.rs`). The function is private
(`fn format_tokens`, not `pub`) in claude-watch and cannot be imported
across crates. The implementation is three lines: `< 1_000` → `"{n}"`,
`< 1_000_000` → `"{:.1}k"`, else `"{:.1}M"`. When `token_total.is_none()`
(parse pending), show no suffix. Tokens are parsed on background thread via
the same `work_tx`/`result_rx` pattern as claude-watch (start with
phase-3 plumbing already in place).

Color: token count in `Color::Cyan`, matching claude-watch.

#### 4.4 Vertical scrollbar on JSONL panel

Port `claude-watch::ui::draw_content_panel`'s scrollbar exactly:

- Compute `max_scroll = total_visual_rows - panel_height` using
  `claude-watch::app::visual_rows` (port the helper into a shared module
  — `vt.rs` is the wrong home, put it in a new `wrap.rs` or inside
  `agent.rs::view`).
- Render a `ratatui::widgets::Scrollbar` with
  `ScrollbarOrientation::VerticalRight` when `max_scroll > 0`.
- Cache `cached_visual_rows: Option<(usize, u16)>` per `AgentView`,
  same invariation rules as claude-watch.

#### 4.5 Phase-4 deliverable check

Manual:

- Kill `claude` in active tab → modal shows `[R][X]`. `R` restarts in
  same cwd. `X` closes the tab.
- Externally `git worktree remove <name>` → tab gets `[removed]` suffix
  within 5s. PTY continues until exit; tab auto-closes when `claude`
  exits.
- Slot labels show `12.3k`-style token counts that match claude-watch's
  display for the same agent.
- Scrolling a long agent session shows a visible scrollbar; arrow keys
  / page keys move both viewport and scrollbar thumb.

## Cross-phase data contracts

These flow across module boundaries; flagging here so the executor does
not change them mid-stream:

- **PTY byte chunk:** `Vec<u8>` from reader thread to main loop. No
  framing assumed; `VtScreen::feed` handles partial sequences (the
  termwiz parser is a state machine).
- **`PtySize`:** always `{ rows, cols, pixel_width: 0, pixel_height: 0 }`.
  `pixel_*` are unused — termwiz does not need them.
- **Agent JSON schema (read-only):** `{ "uuid": "<session-uuid>" }` —
  required. All other fields ignored.
- **Worktree polling cadence:** 5 seconds — locked. Used by both
  worktree and named-agent discovery (single timer drives both).
